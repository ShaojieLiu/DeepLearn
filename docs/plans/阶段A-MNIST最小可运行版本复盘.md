# 阶段A复盘：MNIST最小可运行版本

## 1. 本阶段原目标

`phase A` 的目标很明确：

1. 先把 `MNIST` 跑起来
2. 先建立数据、模型、训练、评估的最小闭环
3. 为后续教材和从零实现准备一个可信基线

## 2. 实际完成情况

本阶段已经完成：

1. `MNIST` 数据可以正常下载、读取和查看
2. `inspect_data.py` 已经验证了数据集大小、样本形状和样本图片保存
3. `train.py` 已经跑通最小训练闭环
4. 测试集准确率已经达到可接受水平
5. 教材已经有了环境准备、数据理解和最小代码说明
6. 实验记录已经单独沉淀到 `book/08-实验记录/`

## 4. 本阶段暴露出来的问题

`phase A` 虽然跑通了，但也暴露出几类真实问题：

1. 教材一开始偏容易写成“规划文档”，而不是“读者可执行材料”
2. 只给理论解释不够，必须给明确命令、脚本名和预期结果
3. 示例代码如果完全没有注释，初学者读起来会断层
4. 术语如果出现得太早，会打断当前章节的学习节奏
5. 实验记录如果混进正文，会明显破坏正文阅读体验

## 5. 这一阶段学到的稳定经验

经过 `phase A`，当前可以确认几条稳定判断：

1. 教材必须围绕真实跑通过的代码写，不能先写大而空的体系
2. 正文、代码、命令、实验记录必须形成证据链
3. 教材中的命令和代码路径，要尽量写到可以直接执行
4. 实验记录应该集中存放，不要分散塞进每一章
5. 教材图片应放到 `book/images/`，不要长期依赖 `data/` 中的实验产物
6. 代码目录应按教学阶段递进，而不是一开始就工程化

## 6. 对整体教材计划的影响

`phase A` 之后，整体教材计划需要做两个明显修订：

1. 近期计划要更收敛
当前不应继续平均推进整本书，而应继续围绕 `MNIST` 完成第一批真正可学、可跟做的章节。

2. 计划结构要更强调“先形成一章完整样板”
相比于把 `CNN`、`ResNet`、`Transformer`、`Diffusion` 都先铺很细，更重要的是先把 `MNIST` 这一章做成可复用模板。

## 7. 下一阶段应该做什么

下一阶段不应立刻跳到 `CNN`。

更合理的下一步是：

1. 补齐 `MNIST` 对应的第一批关键教材章节
2. 把当前已经跑通的基线整理成更适合教学阅读的内容
3. 在此基础上，再进入从零实现版本
