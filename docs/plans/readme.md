# 计划目录说明

本目录用于维护《从零实现到PyTorch：深度学习算法入门与进阶》的执行计划。

与教材正文不同，这里的文件只回答三类问题：

1. 当前先做什么
2. 每个阶段要完成什么
3. 如何判断这一阶段已经完成

## 当前执行方式

当前采用“边学习、边实践、边写教材”的推进方式，不平均展开整本书，而是先完成第一个教学闭环，也就是 `MNIST` 手写数字识别。

当前阶段的核心目标：

1. 先把 `MNIST` 项目真正跑起来
2. 只优先编写与 `MNIST` 直接相关的教材内容
3. 通过“从零实现 + PyTorch 对照”建立第一条完整主线
4. 在 `MNIST` 阶段形成闭环后，再继续扩展到 `CNN`、`ResNet`、`Transformer`、`Diffusion`

## 当前阶段交付范围

### 项目交付物

1. `MNIST` 数据集加载与查看
2. 一个可运行的基线模型
3. 一个从零实现的线性分类器或多层感知机
4. 一个 `PyTorch` 版本的 `MNIST` 分类实现
5. 训练结果、准确率和基本分析

### 教材交付物

1. 教材前言中与当前路线直接相关的章节
2. `MNIST` 卷中的导读与核心章节
3. 与 `MNIST` 项目同步的实践说明
4. 针对当前学习阶段的常见问题总结

## 当前阶段优先级

### 第一优先级

先把 `MNIST` 跑起来，并能完成最基础的训练、验证和结果观察。

### 第二优先级

在项目跑通的基础上，把“为什么这样做”写成教材内容。

### 第三优先级

等 `MNIST` 主线稳定后，再补充更系统的基础准备和后续章节占位。

## 分阶段执行计划

### 阶段 A：建立 MNIST 最小可运行版本

目标是先把项目真正跑起来，而不是一开始追求完整教学体系。

完成标准：

1. 能下载或读取 `MNIST` 数据集
2. 能查看样本图像和标签
3. 能完成一次完整训练
4. 能输出训练集或测试集上的基本指标
5. 能确认整个流程从数据到预测是通的

对应计划文件：

- `docs/plans/阶段A-MNIST最小可运行版本计划.md`

### 阶段 B：补齐 MNIST 对应教材第一批内容

目标是让项目实践和教材内容同步出现，形成第一轮可读材料。

完成标准：

1. 学习者能看懂为什么从 `MNIST` 开始
2. 学习者能理解 `MNIST` 的任务定义和数据结构
3. 学习者能按照教材说明跑通第一个版本

### 阶段 C：完成从零实现版本

目标是通过不依赖深度学习框架的实现，真正理解分类模型的基本工作方式。

完成标准：

1. 能解释输入、权重、输出、损失和梯度分别是什么
2. 能从零实现一个线性分类器或多层感知机
3. 能解释训练循环在做什么

### 阶段 D：完成 PyTorch 对照版本

目标是把已经理解过的底层流程映射到现代主流框架。

完成标准：

1. 能用 `PyTorch` 重写 `MNIST` 分类任务
2. 能理解 `Dataset`、`DataLoader`、`nn.Module`、`optimizer` 分别对应什么
3. 能说明从零实现与框架实现之间的关系

## 当前最小教材输出范围

在 `MNIST` 阶段，优先输出以下内容即可，不必一开始把整本书都展开：

1. 前言卷中和当前路线强相关的 3 到 4 篇文章
2. `MNIST` 卷的导读、任务定义、从零实现、`PyTorch` 对照
3. 当前阶段遇到的关键问题、错误和经验总结

当前真正应该先写好的主线是：

`为什么这样学 -> 为什么从MNIST开始 -> MNIST是什么 -> 怎么从零实现 -> 怎么用PyTorch重写 -> 两者有什么对应关系`
