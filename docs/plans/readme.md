# 计划目录说明

本目录用于维护《从零实现到PyTorch：深度学习算法入门与进阶》的执行计划。

与教材正文不同，这里的文件只回答三类问题：

1. 当前先做什么
2. 每个阶段要完成什么
3. 如何判断这一阶段已经完成

另外有一条统一要求：

每一份阶段计划都必须包含待办清单，方便直接执行和持续更新状态。

## 当前执行方式

当前采用“边学习、边实践、边写教材”的推进方式。

`phase A` 已经完成，当前正式进入 `phase B`。

当前阶段的核心目标：

1. 继续只聚焦 `MNIST`
2. 优先补齐第一批真正可教学、可跟做的教材小节
3. 让正文、代码、实验记录形成稳定回指
4. 在 `MNIST` 这一章形成样板后，再继续扩展到后续模型主题

## 分阶段执行计划

### 阶段 A：建立 MNIST 最小可运行版本

目标是先把项目真正跑起来，而不是一开始追求完整教学体系。

完成标准：

1. 能下载或读取 `MNIST` 数据集
2. 能查看样本图像和标签
3. 能完成一次完整训练
4. 能输出训练集或测试集上的基本指标
5. 能确认整个流程从数据到预测是通的

对应计划文件：

- `docs/plans/阶段A-MNIST最小可运行版本计划.md`
- `docs/plans/阶段A-MNIST最小可运行版本复盘.md`

### 阶段 B：补齐 MNIST 对应教材第一批内容

目标是让已经跑通的第一版结果，被整理成更适合教学阅读的渐进式材料。

完成标准：

1. 学习者能看懂第一版最小代码到底做了什么
2. 学习者能理解为什么教材要按“先跑通、再细化、再深入”的顺序展开
3. 学习者能按照教材说明继续理解训练产物、评估和下一步方向
4. 学习者能顺着当前材料进入后续从零实现阶段

对应计划文件：

- `docs/plans/阶段B-MNIST教材第一批核心内容计划.md`

### 阶段 C：完成从零实现版本

目标是通过不依赖深度学习框架的实现，真正理解分类模型的基本工作方式。

完成标准：

1. 能解释输入、权重、输出、损失和梯度分别是什么
2. 能从零实现一个线性分类器或多层感知机
3. 能解释训练循环在做什么

### 阶段 D：完成 PyTorch 对照版本

目标是把已经理解过的底层流程映射到现代主流框架。

完成标准：

1. 能用 `PyTorch` 重写 `MNIST` 分类任务
2. 能理解 `Dataset`、`DataLoader`、`nn.Module`、`optimizer` 分别对应什么
3. 能说明从零实现与框架实现之间的关系
