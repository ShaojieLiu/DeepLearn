# 阶段B计划：MNIST教材第一批核心内容

## 1. 本阶段目标

`phase B` 不再优先解决“能不能跑通”，而是优先解决：

当前已经跑通的 `MNIST` 基线，能不能被教材稳定地讲清楚，并解释为什么教材要一层一层细化。

完成本阶段后，读者应至少能做到：

1. 看懂最小版本代码在做什么
2. 理解为什么当前先给最小闭环，再逐步细化
3. 按教材指令完成环境准备、数据检查和最小训练
4. 对下一阶段的从零实现形成清晰预期

## 2. 本阶段范围

本阶段仍然只聚焦 `MNIST`。

当前不展开：

1. `CNN`
2. `ResNet`
3. `Transformer`
4. `Diffusion`
5. 大范围补齐基础数学章节

每一篇在本阶段内完成的小节，尽量满足下面几条：

1. 小节职责单一
2. 关键动作有明确命令

5. 当前小节不提前压入后续术语负担

## 6. 任务拆解

### 任务 1：修订总计划

目标：
让总计划反映 `phase A` 已完成、近期重点仍是 `MNIST`。

完成标准：

1. 当前阶段说明已切换到 `phase B`
2. 暂缓事项写清楚
3. 总待办与当前实际情况一致

### 任务 2：补齐 MNIST 第一批核心教材小节

目标：
让学习者能沿着当前代码基线读下去，同时理解为什么教材要按“先跑通、再细化、再深入”的顺序推进。

完成标准：

1. 至少补齐 3 篇关键小节
2. 每篇都能回到当前代码基线
3. 每篇都能明确告诉读者当前先理解什么、暂不展开什么
4. 至少有一节明确解释教材为什么要这样层层细化

### 任务 3：稳定教材和实验记录的关系

目标：
让正文、代码、实验记录之间的回指关系更稳定。

完成标准：

1. 关键小节末尾带实验记录回指
2. 需要图片的地方优先引用 `book/images/`

## 7. 本阶段完成标准

满足下面条件，即可视为 `phase B` 完成：

1. `phase A` 复盘已完成
2. 总计划已修订
3. `MNIST` 第一批核心小节已补齐
4. 正文、代码、实验记录之间已经形成稳定回指

## 8. 本阶段结束后做什么

`phase B` 完成后，下一步进入：

`阶段C：MNIST从零实现版本`

那时重点会转向：

1. 不依赖深度学习框架的最小实现
2. 输入、权重、输出、损失、梯度的直观解释
3. 从当前 `PyTorch` 基线过渡到真正的底层理解
