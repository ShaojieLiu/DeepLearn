# 阶段A计划：MNIST最小可运行版本

本文档只解决一个问题：先把 `MNIST` 手写数字识别任务跑起来，建立第一个可执行闭环。

这一阶段先不追求“从零实现”和“完整教材”，重点是先有一个可运行、可观察、可复现实验结果的最小版本。

## 1. 阶段目标

完成本阶段后，你应该能做到：

1. 成功下载或读取 `MNIST` 数据集
2. 能查看样本图像与标签
3. 能训练一个最小分类模型
4. 能在测试集上得到基础准确率
5. 能知道后续从零实现和教材编写该建立在什么代码基础上

## 2. 本阶段建议策略

建议先做一个最小可运行基线，而不是一开始就手写完整反向传播。

原因是：

1. 先打通数据、模型、训练、评估的全流程
2. 先确认环境、依赖、数据集和代码结构都没有问题
3. 后续再做从零实现时，有一个对照基线

当前阶段建议技术路线：

1. 优先用 `PyTorch` 跑通最小版本
2. 模型先用最简单的全连接分类器或小型 `MLP`
3. 训练时先关注“能跑通”和“能出结果”，不急着调优

## 3. 建议交付物

本阶段建议至少产出以下内容：

1. 一份环境准备说明
2. 一份 `MNIST` 数据加载与查看脚本
3. 一份最小训练脚本
4. 一份测试评估结果
5. 一份运行记录，说明碰到过什么问题、最后是怎么跑通的

## 4. 任务拆解

### 任务 1：准备项目结构

目标：
建立一个适合后续继续扩写的最小项目结构。

建议完成事项：

- [ ] 明确代码主目录放在哪里
- [ ] 规划 `data`、`scripts`、`models`、`notebooks`、`outputs` 是否需要
- [ ] 确定第一个版本的入口脚本文件名

建议产出：

1. 一个最小目录结构说明
2. 一个可直接运行的主脚本

### 任务 2：准备运行环境

目标：
确保依赖安装和运行方式清晰可复现。

建议完成事项：

- [ ] 确认本地 `Python` 版本
- [ ] 安装或确认 `torch`、`torchvision`、`matplotlib` 等依赖
- [ ] 确认使用 `CPU` 还是 `GPU`
- [ ] 记录运行命令

验收标准：

1. 能成功导入核心依赖
2. 能执行训练脚本而不报环境错误

### 任务 3：跑通数据加载

目标：
确认 `MNIST` 数据能正确下载、读取和可视化。

建议完成事项：

- [ ] 下载或自动加载 `MNIST`
- [ ] 打印训练集和测试集大小
- [ ] 读取一批样本
- [ ] 可视化若干张图片与标签
- [ ] 确认输入数据维度和标签范围

验收标准：

1. 能看见手写数字样本图
2. 知道每张图像的 shape、取值范围和标签含义

### 任务 4：实现最小基线模型

目标：
先完成一个最简单可训练的分类模型。

建议完成事项：

- [ ] 选择模型方案
- [ ] 如果求稳，先用单层线性分类器
- [ ] 如果希望稍微好一点，直接用两层 `MLP`
- [ ] 定义损失函数和优化器

推荐：

优先从两层 `MLP` 开始，既简单，又能比较稳定地在 `MNIST` 上得到可接受结果。

验收标准：

1. 模型可以完成前向计算
2. 损失函数可以正常下降

### 任务 5：跑通训练与评估

目标：
确认训练循环、验证流程和评估输出全部可用。

建议完成事项：

- [ ] 实现训练循环
- [ ] 每个 epoch 输出损失和准确率
- [ ] 在测试集上做评估
- [ ] 记录最好成绩和关键参数

建议先用的参数：

1. `batch_size` 从 `64` 或 `128` 开始
2. `epoch` 从 `3` 到 `5` 开始
3. 优化器优先用 `Adam`
4. 学习率先用 `1e-3`

验收标准：

1. 能完整跑完训练
2. 能输出测试准确率
3. 能确认模型确实学到了，而不是随机猜测

### 任务 6：记录结果并沉淀问题

目标：
为后续教材写作准备原始材料。

建议完成事项：

- [ ] 记录最终运行命令
- [ ] 记录关键超参数
- [ ] 记录训练结果
- [ ] 记录踩坑和排错过程
- [ ] 记录后续想优化的问题

建议记录的问题：

1. 数据 shape 是怎样变化的
2. 为什么模型输入需要展平
3. 为什么要有训练集和测试集
4. 为什么损失下降不代表一切

## 5. 建议执行顺序

请按下面顺序执行，尽量不要跳步：

1. 先建目录和主脚本
2. 再装依赖并验证环境
3. 然后跑通数据加载和可视化
4. 接着实现最小模型
5. 再跑训练与评估
6. 最后整理结果与问题记录

## 6. 本阶段完成标准

满足以下条件，即可视为阶段 A 完成：

1. `MNIST` 数据可正常读取
2. 最小模型可正常训练
3. 能输出测试准确率
4. 代码可重复运行
5. 已经沉淀出后续教材可直接使用的原始材料

## 7. 本阶段结束后立刻要做什么

阶段 A 完成后，下一步不要急着上 `CNN`，而是立刻做两件事：

1. 根据实际跑通的过程，编写 `MNIST` 相关教材第一批章节
2. 在当前基线之上，开始做从零实现版本

## 8. 执行时的提醒

1. 先求跑通，不先求优雅
2. 先求结果可信，不先求模型复杂
3. 先求把过程记录下来，不要等做完再回忆
4. 每完成一个小任务，就记录一次观察结果
