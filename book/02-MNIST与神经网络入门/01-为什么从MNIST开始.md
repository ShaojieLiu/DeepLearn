# 为什么从MNIST开始

很多初学者在开始学深度学习时，会有一个自然冲动：既然现在都在讲大模型、`Transformer`、`Diffusion`，为什么还要从 `MNIST` 这种看起来很简单的数据集开始？

答案是：因为你现在最缺的不是复杂模型，而是第一个完整闭环。

## 1. 初学者真正需要的不是复杂，而是闭环

在真正开始做项目之前，很多概念只是名词：

1. 数据集
2. 模型
3. 损失函数
4. 优化器
5. 训练集和测试集
6. 准确率

只有当你真的跑通一次任务，它们才会从抽象名词变成真实对象。

`MNIST` 适合作为第一步，就是因为它能让你以较低成本完成这次第一次闭环。

## 2. 为什么不是一开始就上复杂数据集

如果你一开始就上更复杂的数据集，通常会同时遇到很多问题：

1. 数据更大，下载和预处理更复杂
2. 模型更复杂，结构更难理解
3. 训练更慢，反馈更慢
4. 报错和调试成本更高

这对初学者非常不友好，因为你很难判断：自己卡住，到底是因为概念没懂、代码写错，还是环境或数据出了问题。

`MNIST` 的优势在于，它帮你把很多非必要复杂度先压低了。

## 3. MNIST 的真正价值

`MNIST` 的价值不是“它本身很重要”，而是“它很适合做第一个项目”。

它特别适合帮助你理解：

1. 图像数据进入模型之前长什么样
2. 分类任务的输入和输出分别是什么
3. 训练循环在做什么
4. 损失下降和准确率提升意味着什么
5. 一个模型到底是怎么从不会到会的

## 4. 它也适合作为代码结构的第一阶段

对这个项目来说，`MNIST` 还承担另一个角色：它是你第一套代码结构和教材结构的试验场。

我们现在不想一开始就让 `code/` 变成一个大而全的工程，而是想让它随着学习阶段逐渐生长。

这意味着：

1. `MNIST` 的第一版代码可以非常简单
2. 你可以在它身上观察“什么时候开始需要拆分”
3. 你可以在它身上观察“什么时候开始需要抽象”

这对后面进入 `CNN`、`ResNet`、`Transformer` 都有帮助。

## 5. 从MNIST开始，但不止于MNIST

从 `MNIST` 开始，并不代表整套学习会停留在简单任务上。

更准确地说：

1. `MNIST` 负责建立第一层认知
2. `CNN` 负责建立图像特征提取认知
3. `ResNet` 负责建立深层网络训练认知
4. `Transformer` 负责建立现代架构认知
5. `Diffusion` 负责建立生成模型认知

所以 `MNIST` 不是终点，而是起点。

## 6. 对当前阶段来说，MNIST意味着什么

在当前阶段，选择 `MNIST` 的真正含义是：

1. 先把环境和数据打通
2. 先写一个最小模型
3. 先跑通训练与评估
4. 先建立关于整个流程的第一手体验

这一阶段不需要追求复杂，不需要追求先进，也不需要追求漂亮结构。

你只需要优先确认一件事：

“我能不能把一个完整任务从头到尾跑起来。”

如果可以，说明这一阶段的第一轮目标已经达成。
