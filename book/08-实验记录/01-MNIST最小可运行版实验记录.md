# MNIST最小可运行版实验记录

## 1. 本次实验要验证什么

本次实验只验证一件事：

`MNIST` 的最小训练闭环是否已经跑通。

这里的“跑通”包括：

1. 数据可以正常下载和读取
2. 模型可以完成训练
3. 测试集准确率明显高于随机猜测

## 2. 运行环境

本次实验在项目根目录 `DeepLearn/` 下执行。

依赖环境采用项目虚拟环境：

```bash
source .venv/bin/activate
```

## 3. 运行脚本

本次实验使用的脚本是：

1. `code/01-mnist最小可运行版/inspect_data.py`
2. `code/01-mnist最小可运行版/train.py`

先用 `inspect_data.py` 检查数据链路，再用 `train.py` 跑最小训练闭环。

## 4. 数据检查结果

先执行：

```bash
source .venv/bin/activate
python "code/01-mnist最小可运行版/inspect_data.py"
```

本次实际输出如下：

```text
train size: 60000
test size: 10000
sample shape: (1, 28, 28)
sample label: 5
saved sample image to data/mnist/sample.png
```

这组输出说明：

1. 训练集和测试集都已成功加载
2. 单张样本的形状是 `(1, 28, 28)`
3. 当前样本标签为 `5`
4. 样本图已经成功保存到本地

## 5. 训练命令

运行命令如下：

```bash
source .venv/bin/activate
python "code/01-mnist最小可运行版/train.py"
```

## 6. 关键参数

当前脚本中使用的关键参数如下：

1. `batch_size = 64`
2. `epochs = 3`
3. `learning_rate = 1e-3`
4. 优化器使用 `Adam`
5. 模型是最小两层 `MLP`

## 7. 实际输出结果

本次实际运行输出如下：

```text
epoch=1, test_acc=0.9460
epoch=2, test_acc=0.9622
epoch=3, test_acc=0.9670
```

## 8. 结果说明

这组结果至少说明了三件事：

1. 训练脚本已经可以完整跑完
2. 模型确实学到了有效模式
3. 这版最小基线已经足够作为后续章节的对照版本

当前不追求更高准确率。

这一版的主要目标，是先建立：

1. 数据
2. 模型
3. 训练
4. 评估

这四部分组成的第一层闭环。

## 9. 当前已知边界

这一版已经能训练和评估，但还没有展开下面这些内容：

1. 模型权重如何保存
2. 模型权重如何加载
3. 如何用单张图片做预测
4. 为什么损失函数这样设计
5. 为什么训练模式和评估模式要分开

这些内容会在后续章节中逐步展开，不在当前最小版本里继续堆复杂度。

## 10. 当前结论

`phase A` 的核心目标已经达成：

1. `MNIST` 数据链路已打通
2. 最小训练脚本已跑通

4. 可以进入下一阶段的教材和代码迭代
