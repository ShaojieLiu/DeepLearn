# 本卷导读

这一卷不直接讲模型代码，而是先说明这套教材为什么这样设计、应该怎么使用，以及学习者在当前阶段应该把注意力放在哪里。

对于这个项目来说，教材不是孤立存在的。你会同时看到三条主线一起推进：

1. `book/` 中的教材内容
2. `code/` 中的实战代码
3. `meta/` 中的方法、经验和原则

当前阶段最重要的目标不是“懂完所有理论”，而是先建立第一个最小闭环，也就是 `MNIST` 手写数字识别。

这个最小闭环包括：

1. 理解 `MNIST` 是什么任务
2. 准备好环境
3. 下载并查看数据
4. 跑通一个最小可运行版本
5. 记录结果和问题

因此，阅读顺序建议如下：

1. 先看 [为什么采用从零实现加PyTorch对照](./04-为什么采用从零实现加PyTorch对照.md)
2. 再看 [为什么从MNIST开始但不止于MNIST](./05-为什么从MNIST开始但不止于MNIST.md)
3. 然后进入基础准备卷
4. 最后开始 `MNIST` 卷的第一批章节

当前阶段你不需要一开始就理解 `CNN`、`ResNet`、`Transformer` 和 `Diffusion` 的全部细节。你只需要知道：`MNIST` 是整条学习路径的第一个台阶，它的价值在于帮助你建立关于数据、模型、训练和评估的第一套真实认知。

从这一卷开始，请始终带着一个问题去看后续内容：

“这一章能不能帮助我把当前的代码和理解再往前推进一步？”

如果答案是可以，那它就是当前阶段值得优先阅读的内容。
