# 从零实现到PyTorch：深度学习算法入门与进阶

本文档用于规划整套《从零实现到 PyTorch：深度学习算法入门与进阶》的目录结构与写作组织方式。

## 目录约定

- 本教材所有正文统一放在 `book/`
- 本文件 `readme.md` 只负责维护整套教材的结构规划、写作顺序与目录说明
- 每一卷单独成目录
- 每一章单独成 Markdown 文件
- 文件名统一使用中文
- 章节采用有序编号，便于后续扩写、调整和生成导航

## 教材定位

- 这不是一篇单独的 `MNIST` 教程，而是一套完整的深度学习入门到进阶教材
- `MNIST` 只是第一个上手任务，用来建立数据、模型、损失函数、训练流程和评估方法的最小闭环
- 整套教材后续会继续覆盖 `CNN`、`ResNet`、`Transformer`、`Diffusion`
- 教学方式采用“从零实现 + PyTorch 对照”
- 文风采用“教学讲解 + 项目实战结合”

## 编写原则

### 1. 教材不是孤立存在的

- 本教材不是只讲概念的独立文档集合，而是和对应实战代码一起演进的教学体系
- 教材中的核心章节，原则上都应能在项目代码中找到对应实现、实验脚本或示例
- 代码不是教材的附件，代码本身也是教材的一部分
- 教材写作应优先追求高信息密度，而不是堆积字数
- 教材里能指导重新下载、重新生成的内容，默认不应进入 git
- 教材中凡是要求用户操作的地方，都应尽量给出明确可执行指令
- 教材中的示例代码应补充简洁注释，帮助读者理解每一段的目的和生疏词语的含义

### 2. 文档与代码双线联动

- 教材主文档统一放在 `book/`
- 对应实战代码统一放在 `code/`
- 后续每推进一个阶段，优先保持“代码先跑通，教材再总结，文档再回指代码”
- 重要章节应尽量建立“章节 -> 代码目录 -> 运行脚本 -> 实验结果”的关联关系

### 3. 教材与代码的关系约定

- `code/` 当前优先按教学阶段递进组织，而不是一开始就按最终工程模块组织
- `MNIST` 相关教材章节，优先对应 `code/01` 到 `code/05` 这一组阶段目录
- 工程化目录应后置出现，在学习者已经理解抽象必要性之后再逐步引入
- 当教材结构迭代时，代码目录也应同步调整，避免文档和代码脱节

### 4. 当前阶段执行原则

- 当前阶段优先完成 `MNIST` 项目闭环
- 当前阶段优先编写与 `MNIST` 直接相关的教材内容
- 在 `MNIST` 阶段形成可复用代码结构后，再逐步扩展到后续模型主题
