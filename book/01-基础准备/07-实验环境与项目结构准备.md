# 实验环境与项目结构准备

这一章的目标很直接：把当前阶段需要的环境和目录准备好，让你可以开始动手跑 `MNIST`。

## 1. 当前阶段需要准备什么

在当前阶段，你不需要一开始准备一个完整的大工程。你只需要确保以下几件事成立：

1. 本地有可用的 `Python`
2. 可以安装并导入 `PyTorch`
3. 可以下载或读取 `MNIST`
4. 有一个最小代码目录可以放第一版脚本
5. 有一个共享数据目录可以复用数据集

## 2. 推荐环境

当前阶段推荐如下：

1. `Python 3.10` 或 `Python 3.11`
2. 使用虚拟环境隔离依赖
3. 优先先用 `CPU` 跑通

原因很简单：

1. `MNIST` 足够小，`CPU` 就能完成第一轮实验
2. 先排除显卡环境问题，能减少初学阶段的阻碍
3. 跑通后再考虑 `GPU`，更符合“先闭环、后优化”的原则

## 3. 当前阶段推荐依赖

当前最小依赖建议如下：

1. `torch`
2. `torchvision`
3. `matplotlib`

如果后面需要保存表格结果或做更多分析，再按需补充其他库。

## 4. 推荐安装方式

下面给出一个可以直接执行的最小流程。默认你当前就在项目根目录 `DeepLearn/` 下。

### 方式一：使用 `venv`

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
pip install "numpy<2" torch torchvision matplotlib
```

安装完成后，用下面的命令验证环境：

```bash
python - <<'PY'
import numpy as np
import torch, torchvision, matplotlib

print("torch:", torch.__version__)
print("numpy:", np.__version__)
print(torch.from_numpy(np.array([1, 2, 3])))
print(torch.tensor([1, 2, 3]).numpy())
PY
```

如果这条命令能正常输出版本号，并且 `torch` 与 `numpy` 可以双向转换，说明当前阶段的最小依赖已经准备好了。

### 方式二：如果你使用 Conda

```bash
conda create -n deeplearn python=3.11 -y
conda activate deeplearn
pip install "numpy<2" torch torchvision matplotlib
```

当前阶段只要满足下面三件事即可：

1. `import torch` 成功
2. `import torchvision` 成功
3. `import matplotlib.pyplot as plt` 成功

这里额外强调一下：

如果你直接安装最新版依赖，某些 `torch` / `torchvision` 组合可能会和 `numpy 2.x` 出现二进制兼容问题。典型报错类似：

`A module that was compiled using NumPy 1.x cannot be run in NumPy 2.x`

出现这类问题时，优先先把 `numpy` 约束到 `2` 以下，再重新验证环境。

## 6. MNIST 数据怎么下载

`MNIST` 的经典来源是：

[Yann LeCun - The MNIST Database](http://yann.lecun.com/exdb/mnist/)

在当前项目里，推荐不要手动逐个下载原始文件，而是直接通过 `torchvision` 自动下载到本地目录。

这里补充一点现实情况：

`torchvision` 下载时，可能会先尝试一些历史镜像地址。

所以你有时会先看到某个旧地址返回 `404`，然后再切换到备用地址继续下载。

如果后面备用地址能继续下载，就不算问题。

你现在需要知道的核心点有四个：

1. 第一次运行时会自动下载数据
2. 训练集和测试集要分别加载
3. 图像通常需要转成张量，并在进入全连接模型前展平
4. 下载根目录统一指向 `data/mnist/`

当前建议把下载根目录统一指向：

`data/mnist/`

你可以先手动创建这个目录：

```bash
mkdir -p data/mnist
```

这样后面无论你在 `01`、`02` 还是更后面的 `05` 阶段写代码，都可以复用同一份数据缓存。

如果你现在只想先把数据下载到本地，不想立刻写完整训练脚本，可以直接执行：

```bash
source .venv/bin/activate
python -c "from torchvision import datasets; datasets.MNIST(root='data/mnist', train=True, download=True); datasets.MNIST(root='data/mnist', train=False, download=True)"
```

如果这条命令报错，不要立刻怀疑 `torchvision` 的代码。更常见的原因是本地 Python 的网络证书环境没有准备好。

最常见的两类现象是：

1. 先看到某个旧地址 `404`
2. 然后又看到 `SSL: CERTIFICATE_VERIFY_FAILED`

这通常意味着：

1. 旧镜像地址已经不可用
2. 备用 `HTTPS` 地址本来可以继续下载，但当前 Python 没有找到可信证书，所以握手失败

如果你遇到这种情况，可以按下面顺序处理：

### 第一步：给当前虚拟环境安装证书包

```bash
source .venv/bin/activate
python -m pip install certifi
```

### 第二步：把当前 shell 会话里的 `SSL_CERT_FILE` 指向证书包

```bash
source .venv/bin/activate
export SSL_CERT_FILE="$(python -c 'import certifi; print(certifi.where())')"
```

### 第三步：先验证当前 Python 能访问 `HTTPS`

```bash
source .venv/bin/activate
export SSL_CERT_FILE="$(python -c 'import certifi; print(certifi.where())')"
python - <<'PY'
import urllib.request

with urllib.request.urlopen("https://ossci-datasets.s3.amazonaws.com/mnist/train-images-idx3-ubyte.gz", timeout=20) as r:
    print(r.status)
PY
```

如果这里输出 `200`，说明当前 Python 的 `HTTPS` 校验已经恢复正常。

### 第四步：重新执行 `MNIST` 下载命令

```bash
source .venv/bin/activate
export SSL_CERT_FILE="$(python -c 'import certifi; print(certifi.where())')"
python -c "from torchvision import datasets; datasets.MNIST(root='data/mnist', train=True, download=True); datasets.MNIST(root='data/mnist', train=False, download=True)"
```

如果下载成功，后面训练脚本里继续使用同一个虚拟环境即可。

如果你不想每次都手动执行 `export SSL_CERT_FILE=...`，可以再进一步把这行环境变量配置写进你自己的 shell 配置或虚拟环境激活脚本。

下载完成后，你可以用下面的命令快速确认目录里已经有数据：

```bash
find data/mnist -maxdepth 3 | sed -n '1,20p'
```

当前阶段不必纠结数据底层存储格式，先把“能取到样本、能看到样本、能送进模型”这三步走通。
