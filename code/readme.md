# code 目录说明

`code/` 用于承载《从零实现到PyTorch：深度学习算法入门与进阶》对应的实战代码。

这里的设计原则不是“一开始就搭一个完整工程骨架”，而是让代码结构随着学习阶段逐步生长。

## 核心原则

### 1. 代码结构服务于学习，不先服务于最终工程形态

对于初学者来说，最重要的是先看懂问题、跑通闭环，而不是先面对一个庞大的目录树。

因此，`code/` 优先采用“按教学阶段递进”的组织方式：

1. 第一阶段先用最少文件跑通
2. 后续阶段再根据真实需要逐步拆分
3. 等学习者真正感受到重复、混乱和抽象需求后，再引入工程化结构

### 2. 抽象要后置出现

`common`、`engine`、`metrics`、`configs` 这类抽象并不是开局就必须有的。

在这套教材里，这些目录应该在学习者已经理解“为什么需要抽象”之后再出现，而不是一开始就摆在眼前。

### 3. 每个阶段都应该是可独立理解的

每个阶段目录都尽量只引入该阶段必要的复杂度，让学习者明确看到：

1. 这一阶段新增了什么
2. 为什么要新增这些内容
3. 和上一阶段相比，结构为什么发生了变化

## 当前推荐结构

```text
code/
├── readme.md
├── 01-mnist最小可运行版/
├── 02-mnist加入训练与评估/
├── 03-mnist基础拆分版/
├── 04-mnist从零实现版/
├── 05-mnist-pytorch对照版/
└── 90-工程化整理版/
```

## 各阶段说明

### `01-mnist最小可运行版/`

目标：

1. 用最少代码跑通 `MNIST`
2. 先建立数据、模型、训练、评估的最小闭环
3. 让初学者先看到一个可以工作的最小例子

这一阶段建议：

1. 尽量少文件
2. 尽量少抽象
3. 可以接受代码不够优雅，但必须足够直观

### `02-mnist加入训练与评估/`

目标：

1. 在最小版本基础上加入更明确的训练输出
2. 加入准确率、损失等基础评估信息
3. 让学习者看到“模型不是跑完就结束，还要观察结果”

### `03-mnist基础拆分版/`

目标：

1. 从单文件或少量文件过渡到基础拆分
2. 让学习者理解“为什么开始需要拆分代码”
3. 保持结构仍然足够轻量

这一阶段通常才适合开始引入：

1. `model.py`
2. `train.py`
3. `dataset.py` 或少量工具文件

### `04-mnist从零实现版/`

目标：

1. 用更底层的方式理解前向传播、损失和参数更新
2. 把“框架帮你做了什么”变得清晰
3. 服务于教材中的“从零实现”主线

### `05-mnist-pytorch对照版/`

目标：

1. 用 `PyTorch` 重写已经理解过的任务
2. 把底层概念映射到主流框架
3. 为后续 `CNN`、`ResNet` 等阶段建立统一认知

### `90-工程化整理版/`

目标：

1. 把前面阶段已经验证过的代码结构进一步整理成更完整的工程形式
2. 在学习者已经理解问题之后，再引入更规范的目录设计
3. 为后续多主题扩展提供可复用模板

这里才适合逐步出现：

1. `configs/`
2. `scripts/`
3. `src/`
4. `docs/`
5. `outputs/`
6. `notebooks/`

## 为什么保留 `90-工程化整理版`

这不是为了回到“开局就是大目录”，而是为了保留一个后续收敛入口。

也就是说：

1. `01` 到 `05` 是教学主线
2. `90` 是后期总结性的工程化整理结果

这样既不会在开头吓到初学者，也不会阻断后续扩展到更完整代码仓结构的可能性。

## 教材与代码的映射建议

当前 `MNIST` 相关教材章节，建议优先映射到以下目录：

1. `为什么从MNIST开始` 对应 `01-mnist最小可运行版/`
2. `MNIST数据集与分类任务` 对应 `data/`、`01-mnist最小可运行版/` 和 `02-mnist加入训练与评估/`
3. `训练循环与模型评估` 对应 `02-mnist加入训练与评估/`
4. `从零实现线性分类器` 与 `从零实现多层感知机` 对应 `04-mnist从零实现版/`
5. `使用PyTorch重写MNIST项目` 对应 `05-mnist-pytorch对照版/`
6. `训练工程化与实验管理` 后续再逐步映射到 `90-工程化整理版/`

## 后续扩展建议

当前不建议一开始就在 `code/` 下同时展开 `CNN`、`ResNet`、`Transformer`、`Diffusion` 的完整工程目录。

更合理的方式是：

1. 先把 `MNIST` 的阶段主线走通
2. 总结哪些抽象是真的必要
3. 再把这套阶段式方式复制到下一个主题

例如后续可以继续增加：

1. `11-cnn最小引入版/`
2. `12-cnn基础训练版/`
3. `13-cnn结构拆分版/`
4. `21-resnet入门版/`
5. `31-transformer入门版/`

也就是说，后续主题也应优先延续“教学递进结构”，而不是直接跳到最终工程结构。
