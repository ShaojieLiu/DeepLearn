# meta 目录说明

`meta/` 用于沉淀本项目在“边写教材边学习”过程中的元信息。

这里不放教材正文，也不放主要训练代码，只维护可长期复用的方法层内容。

## 目录结构

```text
meta/
├── readme.md
├── 01-原则/
│   ├── 项目原则.md
│   ├── 教材编写原则.md
│   └── agent沟通原则.md
├── 02-方法/
│   └── 学习与复盘方法.md
├── 03-积累/
│   └── 经验积累.md
└── 04-skill/
    └── skill构想与抽象方向.md
```

## 各目录职责

### `01-原则/`

放稳定原则。

- `项目原则.md`：项目推进的长期原则
- `教材编写原则.md`：教材正文的长期写作规范
- `agent沟通原则.md`：agent 在本项目中的沟通与协作基线

### `02-方法/`

放相对稳定的方法论。

- `学习与复盘方法.md`：如何边做项目边写教材、如何复盘、如何判断什么值得沉淀

### `03-积累/`

放随着项目推进持续增加的经验条目。

- `经验积累.md`：按条目追加已经验证有效的经验

### `04-skill/`

放未来可能抽象为 skill 的方向和草案。

- `skill构想与抽象方向.md`

## 使用原则

1. 原则写进 `01-原则/`
2. 方法写进 `02-方法/`
3. 项目推进中的稳定经验写进 `03-积累/`
4. 未来可能产品化或 skill 化的内容写进 `04-skill/`

这样做的目标是避免重复描述同一件事，也避免把临时想法和稳定规则混在一起。
