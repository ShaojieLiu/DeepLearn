# skill构想与抽象方向

本项目最终不只希望留下代码和教材，还希望逐步形成一套可以辅助用户自主学习的 skill。

## 1. 目标

未来希望把本项目中沉淀出来的方法，抽象成可复用 skill，帮助用户：

1. 规划学习路径
2. 拆解算法项目
3. 边做实验边写总结
4. 在学习过程中持续复盘
5. 用代码、文档和方法论形成闭环

## 2. 可能的 skill 方向

### 方向一：学习路径规划 skill

帮助用户根据当前水平和主题，给出阶段计划、最小闭环和下一步建议。

### 方向二：算法项目拆解 skill

帮助用户把 `MNIST`、`CNN`、`Transformer` 等主题拆成可执行任务。

### 方向三：边学习边写教材 skill

帮助用户把项目过程同步沉淀成教程、笔记和教学文档。

### 方向四：学习复盘 skill

帮助用户把每轮实验、错误和理解沉淀成结构化复盘。

### 方向五：教学型 agent 协作 skill

帮助 agent 在每次对话中同时关注：

1. 当前任务推进
2. 文档沉淀
3. 方法抽象

## 3. 当前阶段不急于定型

现在还不需要立刻写出完整 skill，而是先持续积累原始材料。

现阶段更重要的是：

1. 识别哪些方法稳定有效
2. 识别哪些步骤适合抽象成固定流程
3. 识别哪些沟通原则值得固化

## 4. 后续迭代方式

后续可以持续在这里增加：

1. skill 雏形说明
2. 触发条件
3. 输入输出格式
4. 适用场景
5. 示例工作流
