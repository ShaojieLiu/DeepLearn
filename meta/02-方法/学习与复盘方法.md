# 学习与复盘方法

## 1. 当前采用的学习方法

当前项目采用的学习方式是：

1. 先做项目
2. 再写教材
3. 再复盘方法
4. 再提炼原则

这意味着学习不是线性的“先学完再做”，而是循环式推进：

`实践 -> 解释 -> 复盘 -> 抽象`

## 2. 当前推荐复盘维度

每完成一个阶段，至少从以下角度复盘：

1. 这次到底学会了什么
2. 卡住的地方是什么
3. 哪个解释方式最有帮助
4. 哪个代码结构最适合继续扩展
5. 哪些内容值得写进教材
6. 哪些方法值得抽象成 skill

## 3. 推荐记录格式

每次复盘可以尽量记录：

1. 当前任务
2. 实际操作
3. 遇到的问题
4. 解决方式
5. 关键理解
6. 可复用经验
7. 后续改进点

## 4. 判断一段经验是否值得沉淀

如果某条经验满足以下任意一点，就值得沉淀到 `meta/`：

1. 下次大概率还会用到
2. 对理解算法很关键
3. 能明显提升学习效率
4. 能帮助教材写作更清晰
5. 能转化为更稳定的 agent 行为准则
