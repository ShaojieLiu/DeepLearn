# 教材与写作经验

## 经验 1：教材应追求高信息密度，而不是堆字数

结论：

教材应尽量压缩冗余背景，优先保留关键结论、命令、代码和必要解释。

## 经验 2：教材中凡是要求用户操作的地方，都必须给出明确可执行指令

结论：

教材里只要涉及用户操作，就应尽量给出可以直接执行的命令、路径和文件名。

## 经验 3：环境验证不能只看 import 和版本号，还要验证关键库之间的互操作

结论：

教材中的环境验证，应至少覆盖当前阶段真实会用到的一次关键互操作。

## 经验 4：真实出现过的环境问题，应补到最相关的教材位置

结论：

真实出现过、且读者高概率再次遇到的问题，应写回对应教材位置，附可执行处理步骤。
