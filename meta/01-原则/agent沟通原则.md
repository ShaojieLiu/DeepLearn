# agent沟通原则

本文档用于定义本项目中的 agent 协作与沟通约定。

## 1. 沟通目标

在本项目中，agent 不只是回答问题或改代码，还要帮助维护以下三条主线：

1. 当前学习任务推进
2. 教材内容沉淀
3. 方法论与 skill 方向积累

## 2. 每次对话的默认检查项

后续在本项目中推进任务时，原则上都应先检查：

1. 当前任务属于哪一阶段
2. 这次动作会影响代码、教材还是两者
3. 是否有经验、原则、方法值得写入 `meta/`

## 3. 沟通优先级

1. 先帮助完成当前最重要的学习闭环
2. 再帮助把过程沉淀成文档或计划
3. 再识别是否形成了新的可复用方法

## 4. 输出要求

在本项目中，agent 的输出尽量满足以下要求：

1. 不只给结论，要说明为什么
2. 不只写代码，要说明与教材的关系
3. 不只完成当前任务，要识别是否需要更新 `meta/`

## 5. 持续迭代要求

本文件不是一次性规则，而是项目推进中的沟通基线。

后续如果出现更好的协作方式、复盘方式或沉淀方式，应持续更新这里。

## 6. 关于“每次对话需要调用”的约定

本项目内的后续对话，原则上都应把 `meta/` 视为先验上下文的一部分，优先参考其中的原则、经验和方法。

如果某次推进产生了新的稳定经验，应及时补充到 `meta/`，以便后续对话继续复用。
