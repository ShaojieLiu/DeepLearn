# data 目录说明

`data/` 是整个教学代码共用的数据目录。

它的作用不是承载某一个阶段的代码，而是避免 `01`、`02`、`03`、`04`、`05` 这些阶段反复下载和重复存放同一份数据。

## 设计原则

### 1. 数据目录应按“数据集”划分，而不是按 `train_data` / `test_data` 顶层划分

更合适的方式是：

```text
data/
├── mnist/
├── fashion_mnist/
├── cifar10/
└── ...
```

原因是：

1. 训练集和测试集本来就是某个数据集内部的一部分
2. 后续扩展到多个数据集时，按数据集划分更自然
3. 这样更容易统一管理下载缓存和数据说明

## 当前阶段

当前阶段只需要：

```text
data/
└── mnist/
```

你在 `code/01-mnist最小可运行版/` 到 `code/05-mnist-pytorch对照版/` 中，都应尽量复用这一个共享数据目录。

## 使用建议

当前可以把 `MNIST` 的下载根目录统一指向：

`data/mnist/`

这样做有几个好处：

1. 每个阶段不会重复下载数据
2. 代码目录和数据目录职责更清晰
3. 教材里更容易解释“代码在变，数据目录不变”

## 当前结论

在本项目里，数据目录属于少数适合提前公共化的部分。

因为它解决的是重复下载和重复存放问题，而不是为了追求抽象本身。
