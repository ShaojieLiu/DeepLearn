# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 项目性质

这是一个"边学习深度学习、边实现项目、边编写教材"的长期实验型项目，最终目标是形成三类产物：

1. 完整的深度学习中文教材（`book/`）
2. 可复用的教学型代码仓结构（`code/`）
3. 辅助用户自主学习的 skill 与 agent 协作原则（`meta/`）

## 目录结构

```
book/     # 教材正文，按章/节组织，中文文件名，有序编号
code/     # 实战代码，按教学阶段递进组织（01 → 05 → 90）
docs/     # 计划文件（docs/plans/）
meta/     # 方法论沉淀：原则、方法、经验、skill 构想
```

### `code/` 阶段说明

| 目录 | 教学目标 |
|------|---------|
| `01-mnist最小可运行版/` | 最少代码跑通 MNIST，建立最小闭环 |
| `02-mnist加入训练与评估/` | 加入训练输出和准确率评估 |
| `03-mnist基础拆分版/` | 拆分 `model.py`、`train.py`、`dataset.py` |
| `04-mnist从零实现版/` | 不依赖框架，手写前向传播、损失、参数更新 |
| `05-mnist-pytorch对照版/` | 用 PyTorch 重写，映射底层概念到框架 |
| `90-工程化整理版/` | 工程化整理，含 `src/common`、`src/projects` |

`90-工程化整理版/src/` 是唯一有完整工程抽象的目录，其中：
- `common/`：`data/`、`engine/`、`metrics/`、`utils/`、`visualization/`
- `projects/`：`mnist/`、`cnn/`、`diffusion/`、`resnet/`、`transformer/`

## 核心原则（必须遵守）

### 三线联动原则

每次推进都要同时考虑三条线：
1. **代码**：是否需要新增或修改代码实现
2. **教材**：是否需要同步更新 `book/` 中的教材章节
3. **方法沉淀**：是否有稳定经验应补充到 `meta/`

### 代码结构原则

- **代码结构服务于学习，不先服务于工程形态**：抽象要后置出现，不在早期阶段引入不必要的工程结构
- **每个阶段应独立可理解**：新增代码要能让学习者看清"这一阶段新增了什么、为什么"
- **先闭环，再扩展**：优先建立最小可运行版本，再进入下一主题

### 教材与代码关系

- 教材章节与代码目录必须双向可关联（教材引用代码目录，代码目录在 README 中说明对应章节）
- 不允许文档和代码长期脱节
- 工程化目录（如 `common/`、`configs/`）应在学习者理解"为什么需要抽象"之后才引入

## Agent 沟通约定

每次对话默认先检查：
1. 当前任务属于哪一教学阶段（01~05 还是 90）
2. 这次动作会影响代码、教材，还是两者
3. 是否有经验、原则或方法值得写入 `meta/`

`meta/` 是本项目的先验上下文，每次对话应优先参考其中已沉淀的原则和方法：
- `meta/01-原则/`：项目原则、agent 沟通原则
- `meta/02-方法/`：学习循环与复盘模板
- `meta/03-积累/`：按主题拆分的经验文件
- `meta/04-skill/`：skill 构想方向

输出要求：不只给结论，要说明为什么；不只写代码，要说明与教材的关系。
